# Contributing to Awesome AI Agents for Sales

Thanks for your interest in contributing! This list aims to be the most comprehensive resource for AI tools transforming B2B sales.

## How to Contribute

### Adding a New Tool

1. **Fork** this repository
2. **Add** your tool in the appropriate category
3. **Follow** the existing format
4. **Submit** a pull request

### Format

For tools with pricing:
```markdown
| [Tool Name](https://tool-url.com/) | Brief description (1-2 sentences). Key feature or stat. | Pricing |
```

For open source projects:
```markdown
| [Project Name](https://github.com/org/repo) | Brief description. | Stars |
```

### Examples

Good:
```markdown
| [Gong](https://www.gong.io/) | Market leader in conversation intelligence. 35% higher win rates reported. | ~$100/user/mo |
```

Not ideal:
```markdown
| [Gong](https://www.gong.io/) | Gong is a revenue intelligence platform that uses AI to analyze sales conversations and provide insights to help teams close more deals and improve performance. | Contact sales |
```

## Criteria for Inclusion

### Must Have
- **AI-powered** — Uses machine learning, NLP, or generative AI (not just rule-based automation)
- **Sales-focused** — Directly supports B2B sales workflows
- **Active** — Currently maintained or actively used
- **Accessible** — Has a website or public repository

### Categories
- If a tool fits multiple categories, place it in the most relevant one
- Suggest new categories via issues if needed

### What We Don't Include
- Affiliate or referral links
- Tools that are purely marketing-focused (use [awesome-marketing](https://github.com/marketingtoolslist/awesome-marketing))
- Defunct or abandoned projects
- Tools without clear AI/ML capabilities

## Quality Standards

### Descriptions
- Keep descriptions under 20 words
- Focus on what makes the tool unique
- Include a key stat or differentiator if available
- Use active voice

### Links
- Link to the main product page, not a specific feature page
- For GitHub projects, link to the main repository
- Ensure all links are working

### Pricing
- Use actual pricing if publicly available
- "Contact sales" for enterprise-only pricing
- "Free tier" or "Free - $X/mo" for freemium models

## Pull Request Process

1. **One tool per PR** (unless adding a new category)
2. **Check for duplicates** before submitting
3. **Test your links** to ensure they work
4. **Follow alphabetical order** within categories (where applicable)
5. **Include a brief note** explaining why this tool should be included

## Suggesting Changes

### Issues
Use GitHub issues for:
- Suggesting new categories
- Reporting broken links
- Proposing structural changes
- Discussing controversial additions

### Discussions
Use GitHub Discussions for:
- General questions about the list
- Comparing tools in a category
- Sharing experiences with listed tools

## Code of Conduct

- Be respectful and constructive
- No self-promotion spam
- Disclose any affiliations with tools you're adding
- Focus on helping the community find great tools
